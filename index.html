Projet 1 : Jeu vidéo 2D de plateforme "Ascension"

🎮 Ascension - Jeu de plateforme 2D
Contexte : Projet réalisé dans le cadre du module de programmation orientée objet, en binôme, sur une durée de 6 semaines.
Objectif : Développer un jeu de plateforme 2D complet avec gestion de la physique, des collisions, des niveaux progressifs et une interface utilisateur intuitive.
Technologies utilisées :
  • Langage : C# avec le framework .NET
  • Moteur graphique : utilisation de bibliothèques graphiques pour le rendu 2D
  • Gestion de version : Git pour le travail collaboratif
  • IDE : Visual Studio
Mon rôle : J'ai principalement travaillé sur la logique du jeu et le système de physique. Cela incluait la gestion des mouvements du personnage, la détection des collisions avec les plateformes et les obstacles, ainsi que l'implémentation de la gravité et des sauts.
Fonctionnalités développées :
  • Moteur de physique 2D (gravité, vélocité, accélération)
  • Système de collision pixel-perfect entre le joueur et l'environnement
  • Progression de niveaux avec augmentation de la difficulté
  • Menu principal et système de sauvegarde
  • Gestion des animations du personnage (idle, marche, saut)
Difficultés rencontrées et solutions :
  • Problème : La détection de collision était imprécise, causant des bugs où le joueur traversait les plateformes.
    Solution : J'ai implémenté un système de collision basé sur les boîtes englobantes (bounding boxes) avec vérification des quatre côtés du personnage.
  • Problème : Les mouvements du personnage n'étaient pas fluides à différentes fréquences d'images.
    Solution : Introduction d'un système de delta time pour rendre les mouvements indépendants du framerate.
Résultats : Le projet a été validé avec succès. Le jeu était pleinement fonctionnel avec cinq niveaux jouables. Cette expérience m'a permis de comprendre les enjeux du développement de jeux vidéo et l'importance d'une architecture de code solide.
Compétences développées :
  • Programmation orientée objet avancée (héritage, polymorphisme, interfaces)
  • Gestion d'événements et programmation événementielle
  • Optimisation de code pour les performances temps réel
  • Travail en binôme et résolution collaborative de problèmes

Projet 2 : Émulateur Android en Python

📱 Émulateur Android simplifié
Contexte : Projet personnel réalisé pour approfondir ma compréhension des systèmes d'exploitation mobiles et de l'architecture Android.
Objectif : Créer un émulateur simplifié capable de simuler certaines fonctionnalités de base d'Android, notamment l'interface utilisateur, la gestion des applications et l'interaction avec les composants système.
Technologies utilisées :
  • Langage : Python 3.x
  • Interface graphique : Tkinter pour l'interface utilisateur
  • Gestion des processus : module multiprocessing de Python
  • Manipulation de fichiers : gestion du système de fichiers virtuel
Fonctionnalités implémentées :
  • Interface utilisateur reproduisant l'apparence d'Android (écran d'accueil, barre de navigation)
  • Gestionnaire d'applications permettant d'ouvrir/fermer des applications simulées
  • Système de notifications basique
  • Simulation d'un clavier virtuel
  • Gestion basique des permissions d'applications
Architecture technique : L'émulateur est structuré en plusieurs modules :
  • Module UI : gère l'affichage et les interactions utilisateur
  • Module App Manager : simule le cycle de vie des applications Android
  • Module System : émule les services système (notifications, permissions)
  • Module Storage : gère un système de fichiers virtuel
Défis techniques et apprentissages :
  • Défi : Comprendre l'architecture complexe d'Android pour en reproduire les mécanismes essentiels.
    Apprentissage : J'ai approfondi mes connaissances sur les systèmes d'exploitation mobiles et les concepts d'Activity, Service et Broadcast Receiver.
  • Défi : Gérer plusieurs applications simultanément sans ralentir l'interface.
    Solution : Utilisation du multiprocessing pour simuler l'exécution parallèle d'applications.
Résultats : Bien que simplifié, cet émulateur m'a permis de comprendre en profondeur le fonctionnement d'un système d'exploitation mobile. Le projet reste un work-in-progress que je continue d'améliorer dans mon temps libre.
Compétences développées :
  • Architecture logicielle et conception modulaire
  • Programmation système et gestion des processus
  • Développement d'interfaces graphiques avec Tkinter
  • Documentation technique et code maintenable
